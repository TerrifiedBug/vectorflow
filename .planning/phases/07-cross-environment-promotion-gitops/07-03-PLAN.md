# Phase 07 Plan 03: GitOps Setup Wizard UI

## Objective

Add "Promotion (PR-based)" as a selectable GitOps mode in the environment settings page, with an inline setup guide that walks users through connecting GitHub and configuring the webhook needed for merge-triggered auto-deployment.

## Context

Plan 07-01 implemented the backend: gitops-promotion service, promotion.initiate GitOps branch, and git webhook PR merge handler. The environment router and GitSyncSection component still only support "off" | "push" | "bidirectional" modes — "promotion" is missing from the UI and from the tRPC input schema validation.

## Tasks

### Task 1: Add "promotion" gitOpsMode to environment router

type: auto

Update `src/server/routers/environment.ts`:
- Extend the `gitOpsMode` Zod enum in the `update` procedure to include "promotion"
- When switching TO "promotion" mode and no webhook secret exists, auto-generate a webhook secret (same logic as bidirectional)
- When switching AWAY from "promotion" mode, clear the webhook secret
- Preserve existing bidirectional webhook secret logic unchanged

Done criteria:
- `z.enum(["off", "push", "bidirectional", "promotion"])` in update procedure
- Switching to "promotion" auto-generates encrypted webhook secret if none exists
- Switching away from "promotion" clears webhook secret

### Task 2: Update GitSyncSection with promotion mode and setup guide

type: auto

Update `src/components/environment/git-sync-section.tsx`:
- Add "Promotion (PR-based)" SelectItem for gitOpsMode
- When "promotion" is selected, show an inline setup card with:
  - Step-by-step guide: "1. Save this configuration, 2. In GitHub, go to repo Settings > Webhooks, 3. Add webhook with URL and secret, 4. Select 'Pull requests' event type"
  - Webhook URL display with copy button
  - Webhook secret display (shown once from mutation response, same pattern as bidirectional)
  - "Select 'pull_request' events" instruction (not "push" events like bidirectional)
- Fix the handleSave type cast to include "promotion" in the allowed values

Done criteria:
- "Promotion (PR-based)" appears in the dropdown
- Setup guide appears when "promotion" is selected
- Webhook URL and secret show with copy buttons
- handleSave accepts the "promotion" value without TypeScript error

**Expected state after Tasks 1-2:**
- Environment settings page shows "Promotion (PR-based)" in the GitOps Mode dropdown
- Selecting it shows the webhook setup guide
- Saving generates and reveals a webhook secret
